Where caselaw really comes from

sequenceDiagram
    actor U as User
    participant LP as Library page
    participant Books as Our books
(immigration + ethics)
    participant Chat as Chat
    participant CL as CourtListener
(outside service)

    rect rgb(20, 30, 50)
    Note over U,Books: Scenario 1 — when the user is on the LIBRARY PAGE
    U->>LP: searches for "deportation"
    LP->>Books: so the page asks our shelves
"do we own anything matching this?"
    Books-->>LP: returns immigration items only,
because that's all we own
    LP-->>U: shows a flat list with no caselaw,
and no explanation of why
    Note over LP: Caselaw is invisible here.
The user has no way to know it exists.
    end

    rect rgb(20, 40, 30)
    Note over U,CL: Scenario 2 — when the user is in CHAT
    U->>Chat: asks "what does the law say about deportation?"
    par Chat asks our books
      Chat->>Books: in order to find immigration material we own
      Books-->>Chat: returns immigration items
    and Chat asks CourtListener
      Chat->>CL: in order to fetch live caselaw decisions
      CL-->>Chat: returns real caselaw, just for this question
    end
    Chat->>Chat: merges both source types,
tagging each citation by where it came from
    Chat-->>U: composes one answer with both,
so the user sees caselaw alongside immigration
    end

    Note over CL: Caselaw is borrowed from CourtListener
only at the moment a user asks,
which is why it never appears
on the Library page by default.
