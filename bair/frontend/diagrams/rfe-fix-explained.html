Red path = the old behavior. RFE drafts were treated as a legacy project type and went through the headless-browser render, which never merged your uploaded supporting-document context into the prompt — so the model saw empty {{section_N_info}} placeholders and passed them straight through.
Green path = after PR #1422. RFE drafts (projectTypeId 190) are now routed to the in-process generator, which merges the real extracted context for each section before the model runs.
