BAIR Architecture Plan — Evidence-Based, May 2026

Plan arquitectonico respaldado por 22 fuentes (arxiv + produccion 2025-2026). Tres adiciones criticas sobre v6: evidence gate + claim decomposition (Harvey 0.2% vs Lexis 17-33%), security pass mandatory (Sherlock 2026: 31% vulns en helpers/config), HITL para wrong-requirements no para implementation errors. El diseno de v6 se confirma y fortalece.

Lo que v6 tenia correcto (confirmado por evidencia): Two-commit adversarial — vendor heterogeneity justificado por arxiv 2502.08788. gpt-5.5 SSOT hardcoded. SharedExecutionContext immutable. Verdict FSM. Transport + Publisher por pipeline. Todo se mantiene.

5. Security pass — mandatory hook post-codex

Sherlock Forensics 2026: 31% de vulnerabilidades AI-introduced vienen de helpers, utilities y config tweaks. El hook corre siempre despues de codex_stage, sin opcion de skip.

Security policies son mandatory hooks en el executor, no tools que el agente elige invocar. Un agente no puede ignorar una policy.
